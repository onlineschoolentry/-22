# 보고서 작성용 정리 — Neural ODE + Symbolic Regression 역추론 시스템

> 계획서(Neural ODE + PySR/기호회귀)에 맞춰 **실제 구현된 것**을 정리한 문서.
> 보고서 각 섹션 작성 시 여기서 수치·구조·근거를 가져다 쓰면 됨.

---

## A. 한 줄 정의
카메라로 진자를 찍으면 → 운동을 추적하고 → **Neural ODE로 동역학(벡터필드)을 학습**한 뒤 → **유전 프로그래밍 기호회귀로 명시적 지배방정식을 산출**하는 시스템. 검증 기반 탐구에 "역추론(법칙 도출)" 경험을 더함.

---

## B. 전체 파이프라인 (계획서 4단계 그대로 구현됨)

```
(1) 영상 추적        : 색 기반 트래킹 → 각도 θ(t)        [web/app.js, tracker.py]
(2) 상태 복원        : 운동학 칼만 스무더 → 깨끗한 [θ, ω] [smoother.py]
(3) Neural ODE       : MLP가 벡터필드 f(θ,ω) 학습         [neural_ode.py]
(4) 기호회귀(GP-SR)  : 학습된 f를 명시적 수식으로 변환     [discover_pipeline.py]
        ↓
   θ̈ = -(g/L)·sin(θ) - γ·ω  +  g/L, γ 계수 추출
```

> ※ (2) 스무더는 계획서엔 명시 안 됐지만, Neural ODE에 깨끗한 [θ,ω]를 넣기 위한 전처리. 노이즈 낀 측정에서 ω를 안정적으로 복원해 학습을 도움. (없어도 되지만 실데이터 안정성↑)

## C. 단계별 구현 상세 (보고서 "방법" 섹션용)

### (1) 영상 추적 — θ(t)
- HSV 색 임계화로 마커(추) 검출. **색조(Hue) 좁게, 밝기·채도 넓게** → 스윙 중 조명 변화에도 추적 유지.
- **클릭 색 샘플링**: 마커 클릭 → HSV 중앙값으로 범위 자동 설정 (임의 색 지원).
- **공간 락**: 직전 위치 근방만 탐색 → 같은 색 다른 물체로 안 튐.
- 피벗 기준 `θ = atan2(Δx, Δy)`. (단진자: 한 점 고정 / V자: 두 앵커 중점)

### (2) 운동학 스무더 — 깨끗한 [θ, ω]  (`smoother.py`)
- **등가속도(constant-acceleration) 칼만 + RTS 양방향 스무더.** 물리 가정 없음.
- 단순 수치미분 대비 **가속도 RMSE 14.5배 개선** (5.38 → 0.37, 합성 검증).
- 과정노이즈 q는 로그우도 최대화로 자동 튜닝.

### (3) Neural ODE — 벡터필드 학습  (`neural_ode.py`)
- 상태 `x=[θ, ω]`. 운동학 항등식 `dθ/dt=ω`는 고정, **가속도 `dω/dt=f_NN(θ,ω)`만 MLP가 학습**.
- 구조: **2층 MLP, 은닉 64, tanh 활성** (계획서 명시와 일치).
- 학습: **torchdiffeq의 RK4 적분기**로 적분한 궤적이 관측 [θ,ω]와 일치하도록 MSE 최소화 (Adam).
- 짧은 윈도우(12스텝) 미니배치로 안정 학습, 수백 iter(수분) 내 수렴.
- **물리를 안 알려줬는데 진자 벡터필드를 학습** → 계획서 stage 2 핵심.

### (4) 유전 프로그래밍 기호회귀  (`discover_pipeline.py`)
- 학습된 `f_NN`을 (θ, ω) 그리드에서 샘플링 → 그 (θ, ω, ω̇) 데이터에 **기호회귀** 적용.
- 연산자: `+, -, ×, sin, cos`. 유전 프로그래밍이 후보 수식을 진화시켜 최적 형태 탐색.
- **sin(θ)를 복원** (선형 θ가 아니라) → "왜 이 시스템이 sin 형태인가"를 데이터로 보임.
- 형태 식별 후 **최소자승으로 계수 정밀화** → g/L, γ 정량 추출.

> ⚠️ **방법론 변경(중요, 정직하게 기재)**: 계획서는 **PySR**을 명시했으나, PySR의 **Julia 백엔드가 한글 사용자명(비ASCII Windows 경로)에서 라이브러리 로드 실패**(`sys.dll` 로드 불가)로 실행 불가했음. 따라서 **동일 방법론인 유전 프로그래밍 기호회귀 라이브러리 `gplearn`(순수 Python)으로 대체**. PySR과 gplearn은 둘 다 GP 기반 기호회귀로, **방법론은 동일**하고 라이브러리만 다름. → 보고서엔 "환경 제약으로 gplearn 사용, 방법론(GP-SR) 동일"로 기재 권장.

## D. 검증 결과 (보고서 "결과" 섹션용)

### D-1. 합성 데이터 (정답을 아는 감쇠진자, end-to-end)
| 항목 | 결과 |
|---|---|
| Neural ODE 벡터필드 복원 RMSE | 신호의 **2.6%** |
| Neural ODE가 읽은 g/L | 14.07 (참값 14.014, **0.4%**) |
| Neural ODE가 읽은 γ | 0.259 (참값 0.25) |
| **전체 파이프라인** GP-SR 발견 형태 | `-13.18·sin(θ) - (감쇠항)` → **sin 복원** |
| 계수 정밀화 g/L | 13.88 (**0.95% 오차**) |
| 계수 정밀화 γ | 0.236 (5.5% 오차) |

→ **물리 모르고 출발해서 sin(θ) 형태 + g/L·γ를 정확히 도출.** 알고리즘 정확성 입증.

### D-2. 실제 진자 데이터 — 세 방법이 모두 일치 (핵심 결과)
- 단진자, 최대 진폭 **51°**, 60fps. (V자 진자 → 단진자 전환으로 큰 진폭 확보)
- 큰 진폭 앞구간(42°)으로 Neural ODE 학습 (실데이터는 노이즈+감쇠라 lr 낮추고 1500 iter).

**g/L을 세 가지 독립 방법으로 산출 → 모두 일치:**
| 방법 | g/L | 비고 |
|---|---|---|
| **Neural ODE + GP-SR** | **38.96** | GP가 sin(θ) 지배항 복원 |
| STLSQ (SINDy) baseline | 39.9 | `θ̈ = -39.9·sin(θ)` |
| 주기 측정 `(2π/T)²` | 38.2 | 영상 주기 T=1.0s |

→ **AI 역추론(Neural ODE+GP-SR), 희소회귀(STLSQ), 고전적 주기측정 — 셋이 ~38~40으로 일치.**
이것이 계획서 기대효과 2(해석가능성·교차검증)의 직접 증거. **단순 데이터 피팅이 아니라 동역학 구조를 잡았음**을 세 방법 일치로 입증.

### D-3. 교차 검증 (계획서 기대효과 2)
- 도출식 계수에서 g/L 추출 → **주기 측정(`T=2π√(L/g)`)으로 독립 산출한 g/L과 비교.** 실데이터 4% 일치.
- 즉 **두 독립 방법(동역학 회귀 vs 주기)이 같은 g/L** → 역추론 타당성 확인.

### D-4. baseline 교차검증 (강점)
- 같은 데이터에 **Neural ODE+GP-SR** 과 **직접 구현 STLSQ(SINDy)** 두 방법을 적용 → 결과 일치.
- "두 독립적 발견 방법이 같은 방정식" → 결과 신뢰도 보강.

### D-5. 일반화 검증 (계획서 기대효과 3 — 직접 입증)  [그림: `generalization.png`]
- **θ0=1.0 rad 하나의 궤적만 학습**한 Neural ODE로, **학습에 안 쓴 초기조건**(23°, 40°, 46°)에서 궤적 예측:
  | 안 본 초기조건 | θ(t) 예측 RMSE |
  |---|---|
  | 23° | 0.17° |
  | 40° | 0.12° |
  | 46° | 0.23° |
- 안 본 조건에서도 예측이 실측과 거의 일치 → **특정 궤적을 암기한 게 아니라 동역학(벡터필드)을 학습**했음을 입증.
- 그림: 학습된 벡터필드(quiver) 위에 true vs Neural ODE 예측 궤적 겹쳐 그림 + θ(t) 비교. → **"과학적 모델 vs 단순 데이터 피팅"의 차이를 직접 보여주는 핵심 그림.**

## E. 핵심 통찰 (보고서 "고찰"용)
1. **노이즈의 2차 미분이 발견을 죽인다** → 스무더(또는 Neural ODE의 궤적 학습)가 깨끗한 동역학을 확보하는 게 핵심.
2. **식별 가능성은 실험 설계 문제** → 작은 진폭(<10°)에선 `sin(θ)≈θ`라 선형/비선형 구분 불가(`corr(θ,sinθ)=1.0`). **큰 진폭(30~50°)이어야** GP-SR이 sin을 선형 위로 골라냄.
3. **측정 기하** → 단일 카메라는 영상 평면 내 각도만 측정. 스윙 평면을 카메라와 평행하게 정렬해야 함. (V자 진자는 구조물 가림/평면 정렬 문제로 단진자로 전환)
4. **일반화 가능성**(계획서 기대효과 3) → 학습 안 한 초기조건에서 Neural ODE 궤적 예측을 실측과 비교 → 단순 피팅이 아닌 동역학 학습임을 확인. *(검증 추가 가능)*

## G. 설계 결정 및 정직한 한계 (5개 항목 처리)

**1. 웹 앱에 Neural ODE 발견 연결됨 (로컬 모드)**
- **로컬 실행**(`python web_server.py --http`, torch 설치된 PC)에서 웹 버튼 **"Neural ODE 발견"** → 백엔드 `/api/discover_neural`가 풀 파이프라인 실행 → 브라우저에 **방정식 + 3방법 g/L 교차검증** 표시.
- 즉 **브라우저 안에서 클릭 한 번으로 Neural ODE+GP-SR 발견**까지 됨 (학습 1~2분 대기).
- 웹 "Fit Model"(STLSQ)은 빠른 미리보기로 병존 → 두 방법 비교 가능.
- ⚠️ **Render 배포본은 torch 미설치라 Neural ODE 버튼 안 됨**(STLSQ만). → **발표는 로컬 PC로** 하면 전 과정 브라우저에서 시연 가능.
- 오프라인 `run_demo.py`도 그대로 사용 가능(그림까지 출력, 무대 안정성↑).

**2. GP 원시출력 정리 (처리됨)**
- gplearn 원시식은 항이 많음 → **sympy로 simplify + 최소자승 계수정밀화** → 발표/보고서엔 깔끔한 `θ̈ = -39.0 sin(θ) - 0.01 ω` 로 제시. (`run_demo.py`가 이 형태로 출력)

**3. PySR → gplearn (처리됨, 정직 기재)**
- PySR의 Julia 백엔드가 **한글 사용자명(비ASCII Windows 경로)에서 `sys.dll` 로드 실패**. → 동일 방법론(유전프로그래밍 기호회귀)인 **gplearn(순수 Python)**으로 구현. 방법론 동일, 라이브러리만 다름.

**4. 진자 외 실험 (스코프 결정)**
- 자유낙하·빗면은 **빠른 운동(블러·프레임부족)·마찰·구름관성·슬로모 타이밍 정밀도** 문제로 정밀 측정 난이도 높음(갈릴레오가 빗면 쓴 이유). → **진자를 주 실험으로 집중**. 진자만으로 검증·발견·교차검증·일반화 모두 충족.

**5. 발표 자료 (작성됨)**
- `발표_슬라이드.md` — 11장 슬라이드 구성 + 라이브 데모 시나리오 + 예상 Q&A.

**그 외 본질적 한계**
- 단일 카메라 → 평면 운동 전용 (3D는 스테레오 필요).
- 작은 진폭(<10°)에선 sin≈θ라 비선형 발견 불가 → 큰 진폭(40°+) 필요(식별 가능성).
- 브라우저 추적 노이즈로 미세 비선형 검출엔 한계.

## H. 보고서에서 강조할 포인트
1. **"검증 → 발견"**: 같은 진자에서 (검증) 주기로 g 확인 + (발견) 데이터에서 방정식 도출.
2. **Neural ODE가 물리 없이 벡터필드 학습** → 블랙박스지만 정확(합성 0.4%).
3. **GP-SR이 sin(θ)를 선택** → 해석 가능, "왜 sin인가"를 데이터로.
4. **g/L·γ 정량 추출 + 주기 교차검증 4% 일치** → 역추론 타당성.
5. **AI for Science 방법론을 고교 실험에 이식** (Neural ODE 2018, Symbolic Regression).
6. (정직) **PySR→gplearn 대체**는 환경 제약 때문, 방법론(GP-SR) 동일.
